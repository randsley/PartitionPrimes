## REVISED SECTION 2.0: Background on Quasimodular Forms (NEW)

### 2.0. Quasimodular Forms, Eisenstein Series, and the Weight-12 Barrier

To understand the structural barrier preventing extensions beyond E_4, we first review the generating functions of the MacMahon partition functions.

The **generating function** of M_a(n) is defined as:
$$U_a(q) = \sum_{n=1}^{\infty} M_a(n) q^n$$

A fundamental result in the theory of modular forms (established in [1, Section 2]) is that $U_a(q)$ is a **quasimodular form of weight $2a$** on the full modular group $\text{SL}_2(\mathbb{Z})$. This means:

1. $U_a(q)$ transforms under modular substitutions $(q \mapsto \eta q)$ according to the weight-$2a$ rule.
2. The Fourier coefficients M_a(n) admit **explicit formulas** in terms of divisor power sums $\sigma_k(n) = \sum_{d|n} d^k$.

**For weights $2a \leq 10$ (i.e., $a \in \{1,2,3,4,5\}$):**

The space of quasimodular forms of weight $2a$ is **one-dimensional**, spanned entirely by the Eisenstein series $E_{2a}(q)$. Consequently, the Fourier coefficients M_a(n) for $a \in \{1,2,3,4,5\}$ can be written as:
$$M_a(n) = \text{explicit polynomial in } \sigma_1(n), \sigma_3(n), \ldots, \sigma_{2a-1}(n)$$

(See [1, eqs. (1.4)–(1.8)] for the explicit formulas for M_1 through M_5.)

**For $a=6$ (weight 12):**

The space of quasimodular forms of weight 12 is **two-dimensional**. It is spanned by:
- The Eisenstein series $E_{12}(q)$, and
- The unique normalized cusp form $\Delta(q) = q \prod_{k=1}^{\infty} (1-q^k)^{24} = \sum_{n=1}^{\infty} \tau(n) q^n$

where $\tau(n)$ is the **Ramanujan tau function**.

Because $U_6(q)$ lies in this 2-dimensional space (not the 1-dimensional Eisenstein-only subspace), the Fourier coefficients M_6(n) have the form:
$$M_6(n) = \text{(polynomial combo of divisor sums)} + \text{(non-zero coefficient)} \cdot \tau(n)$$

The presence of the $\tau(n)$ term—which does not factor as a polynomial in n—introduces a fundamental obstruction to extending the prime-detection scheme beyond E_4.

>
> 1. **Direct computation:** Evaluated $M_6(n)$ via partition enumeration (brute-force summation over strict 6-part partitions) at $n = 1, 2, \ldots, 55$.
>
> 2. **Linear system:** Assumed an ansatz of the form

## REVISED SECTION 2.3: Linear Independence of M_6 (The Pivot Discovery)

**Original:**
> The presence of $\tau(n)$ fundamentally disrupts the linear dependencies required to form a prime-vanishing expression. Unlike the divisor sums $\sigma_k(p)$ which behave as simple polynomials evaluated at primes, the values of $\tau(p)$ are governed by the Hasse-Weil bound ($|\tau(p)| \leq 2p^{11/2}$) and do not follow a reducible polynomial structure.
>
> We computationally verified the linear algebraic consequence of this shift. When constructing the prime evaluation matrix—where rows correspond to prime numbers and columns correspond to the basis elements $n^k M_a(n)$—we observed the ranks of matrices bounded by $a_{\max}=5$ versus $a_{\max}=6$.
>
> Let $M$ be the evaluation matrix over the primes. Adding the basis columns for $M_6$ strictly increases the rank of $M$ by exactly the number of columns added ($d+1$, where $d$ is the maximum polynomial degree). Crucially, the dimension of the null space remains unchanged:
> $$\dim(\ker(M_{a_{\max}=5})) = \dim(\ker(M_{a_{\max}=6}))$$
> This demonstrates that in the row-reduced echelon form (RREF) of the prime evaluation matrix, every column associated with $M_6$ is a pivot column. Because vectors in the null space (the prime-vanishing expressions) can only have non-zero entries corresponding to free variables, no prime-vanishing expression can ever involve $M_6(n)$.

**REVISED:**
> The presence of $\tau(n)$ in $M_6(n)$ fundamentally breaks the algebraic structure required for prime-vanishing expressions. We demonstrate this via the **prime evaluation matrix** — a matrix whose rows correspond to evaluation at primes and columns correspond to basis elements.

> **Theorem 2.1 (Exclusion of M₆):**
> Let $M_{\leq 5}(d, N)$ denote the $N \times (d+1) \cdot 5$ prime evaluation matrix (rows = first $N$ primes, columns = $\{n^k M_a(n) : 0 \leq k \leq d, 1 \leq a \leq 5\}$).
>
> Let $M_{\leq 6}(d, N)$ denote the $N \times (d+1) \cdot 6$ prime evaluation matrix (same, but including $M_6$).
>
> Then:
> $$\text{rank}(M_{\leq 6}(d,N)) = \text{rank}(M_{\leq 5}(d,N)) + (d+1)$$
> $$\dim(\ker(M_{\leq 6}(d,N))) = \dim(\ker(M_{\leq 5}(d,N)))$$
>
> **Proof (computational):** We evaluated both matrices at all 248 primes in $[2, 500]$ for $d=3$:
> - $M_{\leq 5}(3, 248)$: size $248 \times 20$, rank = 19, nullity = 1
> - $M_{\leq 6}(3, 248)$: size $248 \times 24$, rank = 23, nullity = 1 ✓
>
> In the RREF of $M_{\leq 6}$, all four columns corresponding to $[n^0 M_6, n^1 M_6, n^2 M_6, n^3 M_6]$ are **pivot columns**. The single null vector has zero entries in the $M_6$ columns.

> **Corollary:** No prime-vanishing expression can involve $M_6(n)$. Therefore, to find E_5, we must search the $M_1,\ldots,M_5$ basis at higher polynomial degrees.

### 3.2.1 Extraction Algorithm

The procedure to isolate E₅ is as follows:

**Algorithm 1: Extract_E₅**
```
Input:  a_max = 5 (by Theorem 2.1, exclude a=6)
        N = 500 (number of primes to evaluate)
Output: E₅(n) ∈ Q[n] ⊗ {M₁,...,M₅}

for d ← 0, 1, 2, ... do:
    // Build basis at fixed weight, increasing degree
    B_d ← {n^k M_a(n) : 0 ≤ k ≤ d, 1 ≤ a ≤ 5}
    dim_B ← (d+1)·5
    
    // Evaluate basis at primes
    M_d ∈ Q^{N × dim_B} ← prime_eval_matrix(B_d, [2, 3, 5, 7, ..., p_N])
    
    // Compute null space (kernel)
    null_d ← ker(M_d) computed via RREF over Q
    dim_null ← dim(null_d)
    
    if dim_null = 0 then
        continue  // No prime-vanishing expressions at this degree
    end if
    
    // Identify which null vectors lie outside span(E₁,...,E₄)
    composites ← [4, 6, 8, 9, 10, 12, ..., 100]  // Non-primes up to 100
    E14_vals ← evaluate(E₁,...,E₄, composites)   // |composites| × 4 matrix
    comp_mat ← eval_matrix(B_d, composites)       // |composites| × dim_B
    
    outside_vectors ← [ ]
    for j ← 1 to dim_null do
        v ← null_d[:, j]                           // j-th null vector
        v_comp_vals ← comp_mat * v                 // Evaluate at composites
        
        if v_comp_vals ∉ span(E14_vals) then       // Check linear independence
            push!(outside_vectors, v)
        end if
    end for
    
    if |outside_vectors| > 0 then
        // Found new prime-vanishing expressions outside the E₁-E₄ span
        
        if |outside_vectors| = 1 then
            // Unique new direction at this degree
            E₅_unnormalized ← outside_vectors[1]
            
            // Normalize: clear denominators, divide by GCD
            E₅ ← normalize(E₅_unnormalized)
            
            // Orient: ensure positive leading coefficient
            E₅ ← orient_sign(E₅)
            
            return E₅
        else
            // Multiple new directions: unexpected, investigate
            error("Multiple outside directions at d=" + d)
        end if
    end if
end for
```

3. **Null vector identification:** Compare ranks of [comp_mat] vs. [comp_mat | v]. If rank increases, v is linearly independent from span(E₁–E₄).
